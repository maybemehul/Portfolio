DSA Learnings

Linked List

A data structure where nodes link to each other in sequence. Singly linked lists connect one way, doubly linked lists connect both ways, and circular linked lists loop back to the start.

Learn more

Stacks

A LIFO structure with push and pop operations. It can be implemented using arrays (fixed size) or linked lists (dynamic size).

Learn more

Queues

A FIFO structure where elements are added at the rear and removed from the front, implemented with arrays (fixed) or linked lists (flexible).

Learn more

Heaps

A tree where the largest or smallest element is at the root, depending on if it’s a max or min heap, ideal for priority-based tasks

Learn more

Binary trees

A structure where each node has up to two children. Complete binary trees fill levels from left to right, balanced trees keep subtrees almost equal in height, and perfect binary trees are fully filled and symmetrical.

Learn more

Binary Search trees

A binary tree with sorted nodes, allowing efficient search, insert, and delete operations.

Learn more

Graphs

A set of nodes connected by edges, representing relationships. It can be directed or undirected, weighted or unweighted, and is useful in networks.

Learn more

Complexities

Measures time and memory an algorithm requires. Time complexity shows runtime growth; space complexity shows memory usage, both expressed in Big O notation.
